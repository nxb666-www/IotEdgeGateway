# 项目架构说明

## 1. 项目定位

IotEdgeGateway 是基于 RK3568 边缘节点和 STM32 下位机的物联网智能控制系统。系统完成传感器数据采集、执行器控制、规则联动、历史数据存储、视频预览、Web/Qt 展示等功能。

项目整体分为四层：

```text
传感器/执行器层  ->  通信接入层  ->  RK3568 边缘网关层  ->  Web/Qt 展示控制层
```

其中 STM32 负责直接连接传感器和外设，ESP8266 负责 WiFi 通信和 MQTT 接入，RK3568 负责本地消息转发、设备状态维护、规则处理、数据库存储和 Web 服务。

## 2. 总体架构

```text
STM32F411
  ├── AHT30 温湿度 (I2C)
  ├── 光照传感器 (ADC)
  ├── 红外检测 (GPIO)
  ├── LED (PWM)
  ├── 电机/风扇 (PWM)
  └── 蜂鸣器 (GPIO)
        │
        │ UART (AT指令)
        ▼
ESP8266 WiFi 模块
  ├── 连接 WiFi 热点
  ├── 连接 MQTT Broker (192.168.x.x:1884)
  ├── 订阅 iotgw/dev/cmd/# (接收控制命令)
  └── 发布 iotgw/dev/telemetry/* (上传传感器数据)
        │
        │ MQTT over WiFi
        ▼
RK3568 边缘节点
  ├── iotgw_mqtt_broker     本地 MQTT Broker (端口 1884)
  ├── iotgw_gateway         主网关程序 (端口 8081)
  ├── Mongoose              HTTP / WebSocket / MQTT 基础库
  ├── SQLite                历史数据存储
  ├── 规则引擎              自动控制与告警
  ├── V4L2 + FFmpeg/GStreamer 视频模块
  └── Web 静态页面 (www/index.html)
        │
        ├── HTTP REST API
        ├── WebSocket 实时推送
        └── Qt 客户端访问相同 API
```

## 3. 主要模块

| 模块 | 位置 | 职责 |
| --- | --- | --- |
| 主程序 | `src/gateway` | 启动配置、日志、设备注册、MQTT、HTTP、规则、SQLite、视频接口 |
| MQTT 客户端 | `src/core/device/protocol_adapters/mqtt_adapter` | 连接本地 Broker，订阅遥测主题，发布控制命令 |
| MQTT Broker | `src/tools/mqtt_broker` | 基于 Mongoose 的轻量 MQTT Broker，监听 `1884` |
| 设备注册表 | `src/core/device` | 保存设备 ID、类型、主题、在线状态、最新 payload |
| REST API | `src/services/web_services/api` | 提供设备查询、控制、规则、历史、摄像头接口 |
| WebSocket | `src/services/web_services/websocket` | 向浏览器实时广播 MQTT 数据 |
| 规则引擎 | `src/core/control` | 根据传感器阈值自动发布执行器命令 |
| SQLite 存储 | `src/core/storage` | 保存遥测历史数据，支持 `/api/history` 查询 |
| 视频模块 | `src/services/web_services/api/camera_api.cpp` | 调用 V4L2 + FFmpeg/GStreamer 生成实时 JPEG、拍照、录制 |
| Web 页面 | `www/index.html` | 实时状态显示、外设控制、历史查询、视频控制 |
| Qt 客户端 | `qt_client/` | 与 Web 功能一致的桌面客户端 |
| 配置文件 | `config` | 环境、设备、规则、存储、网络配置 |
| STM32 固件 | `F411/` | 传感器采集、外设控制、ESP8266 通信 |
| 部署脚本 | `scripts/` | 编译脚本 build.sh、打包脚本 package_iotgw.sh |

## 4. 数据上行链路（传感器 → Web/Qt）

```text
STM32 读取传感器 (AHT30/光照/红外)
  -> 组装 JSON: {"device_id":"temp","type":"sensor","data":{"value":27.7},"ts":0}
  -> 通过 UART 发送给 ESP8266
  -> ESP8266 发布 MQTT 到 iotgw/dev/telemetry/temp
  -> iotgw_mqtt_broker 转发
  -> iotgw_gateway 收到消息
  -> 更新 DeviceRegistry (内存中的设备状态)
  -> 写入 SQLite (历史数据)
  -> 规则引擎判断是否触发
  -> WebSocket 广播给所有客户端
  -> Web/Qt 页面每 1 秒轮询 /api/status 刷新显示
```

MQTT 遥测主题：

```text
iotgw/dev/telemetry/temp     温度
iotgw/dev/telemetry/humi     湿度
iotgw/dev/telemetry/light    光照
iotgw/dev/telemetry/ir       红外 (0=安全, 1=有人)
```

遥测 payload 格式：

```json
{
  "device_id": "temp",
  "type": "sensor",
  "source": "stm32",
  "driver": "aht30",
  "data": {
    "value": 27.7
  },
  "ts": 0
}
```

## 5. 控制下行链路（Web/Qt → 执行器）

```text
Web/Qt 用户点击按钮
  -> POST /api/control
     {"type":"control","payload":{"led_on":1,"led_br":80}}
  -> iotgw_gateway 提取 payload
  -> MQTT 发布到 iotgw/dev/cmd/control
  -> iotgw_mqtt_broker 转发
  -> ESP8266 收到消息 (订阅了 iotgw/dev/cmd/#)
  -> ESP8266 通过 UART 转发给 STM32
  -> STM32 解析 JSON，提取控制字段
  -> 执行对应操作 (LED/电机/蜂鸣器)
```

控制 payload 格式：

```json
{
  "type": "control",
  "payload": {
    "led_on": 1,
    "led_br": 80,
    "motor_on": 1,
    "motor_sp": 50,
    "motor_dir": 0,
    "buzzer": 0
  }
}
```

字段说明：

| 字段 | 含义 | 取值 |
|------|------|------|
| led_on | LED 开关 | 0=关, 1=开 |
| led_br | LED 亮度 | 0-100 |
| motor_on | 电机开关 | 0=关, 1=开 |
| motor_sp | 电机速度 | 0-100 |
| motor_dir | 电机方向 | 0=正转, 1=反转 |
| buzzer | 蜂鸣器 | 0=关, 1=开 |

## 6. Web 与 Qt 交互方式

Web 和 Qt 客户端都不直接访问 STM32，也不直接访问传感器硬件，而是统一访问 RK3568 上的网关服务。

```text
Web/Qt -> HTTP REST API -> iotgw_gateway -> DeviceRegistry / SQLite / MQTT
Web/Qt <- HTTP 轮询 (每1秒) <- iotgw_gateway <- 最新设备状态
```

API 接口：

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| GET | `/api/version` | 版本信息 |
| GET | `/api/devices` | 设备列表 |
| GET | `/api/devices/:id` | 设备详情 |
| GET | `/api/status` | 传感器数据聚合 |
| POST | `/api/control` | 下发控制命令 |
| POST | `/api/actuators/:id/set` | 执行器控制 |
| GET | `/api/rules` | 规则列表 |
| POST | `/api/rules/reload` | 重载规则 |
| GET | `/api/history` | 历史数据查询 |
| GET | `/api/camera/status` | 摄像头状态 |
| POST | `/api/camera/start` | 启动推流 |
| POST | `/api/camera/snapshot` | 抓拍 |

## 7. 视频模块架构

摄像头属于 RK3568 本地外设，视频数据不经过 STM32。

```text
CSI/USB Camera
  -> /dev/videoX
  -> V4L2
  -> FFmpeg 优先 / GStreamer 兜底
  -> /tmp/rk_stream/live.jpg
  -> GET /stream/live.jpg
  -> Web/Qt 显示
```

## 9. 通信协议

### 9.1 STM32 <-> ESP8266

STM32 通过 UART 发送 AT 指令控制 ESP8266：

```text
AT+CWJAP="nie","12345678"           # 连接 WiFi
AT+MQTTCONN=0,"192.168.x.x",1884,1 # 连接 MQTT Broker
AT+MQTTSUB=0,"iotgw/dev/cmd/#",0   # 订阅命令主题
AT+MQTTPUBRAW=0,"topic",len,0,0    # 发布消息
```

### 9.2 MQTT 主题规范

```text
遥测上行: iotgw/dev/telemetry/{device_id}
控制下行: iotgw/dev/cmd/control
```

### 9.3 JSON 数据格式

传感器数据：

```json
{"device_id":"temp","type":"sensor","source":"stm32","driver":"aht30","data":{"value":27.7},"ts":0}
```

控制命令：

```json
{"type":"control","payload":{"led_on":1,"led_br":80,"motor_on":1,"motor_sp":50,"motor_dir":0,"buzzer":0}}
```

## 10. 关键设计说明

1. MQTT Broker 放在 RK3568 本地，方便边缘节点离线运行。
2. 网关程序使用 Mongoose 同时支撑 HTTP、WebSocket 和 MQTT client。
3. SQLite 只存遥测历史数据，不存控制命令。
4. Web/Qt 都通过同一套 REST API 访问网关，保证数据一致。
5. 视频模块属于 RK3568 本地能力，和 STM32 控制链路分离。
6. start.sh 内置 WiFi 自动连接，部署时只需拷贝一个目录即可运行。
7. 规则引擎支持自动化控制和告警，可动态启停。

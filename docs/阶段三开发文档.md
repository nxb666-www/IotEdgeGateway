# 阶段三开发文档：Web 前端 + WebSocket 实时推送

## 一、我做了什么（代码改动）

### 1. POST /api/control 接口（rest_api.hpp）
**文件：** `src/services/web_services/api/rest_api.hpp`

实现了前端控制命令下发接口。浏览器点按钮后发 HTTP POST 请求，网关解析 JSON 提取每个执行器的参数，通过 MQTT 发布到对应主题。

**处理流程：**
```
浏览器 POST /api/control
  → 解析 JSON: {"led_on":1,"led_br":80,"motor_on":1,...}
  → 查注册表找每个执行器的 command_topic
  → 通过 MQTT 发布命令到 Broker
  → 返回 {"status":"正常"}
```

**新增函数：**
- `HandleControl()` — 解析控制命令，通过 MQTT 下发
- `GetJsonInt()` — 从 JSON 字符串提取整数值
- `ExtractValue()` — 从设备数据中提取传感器值
- `ExtractNumber()` — 提取数字字段

### 2. WebSocket 实时推送（gateway_core.cpp）
**文件：** `src/gateway/gateway_core.cpp`

MQTT 收到消息后，除了更新注册表，还通过 WebSocket 广播给所有浏览器客户端。

**新增逻辑：**
```
MQTT 消息到达
  → 更新设备注册表
  → WebSocket 广播: {"type":"mqtt_msg","topic":"...","payload":"..."}
```

WebSocket 收到浏览器消息后，通过 MQTT 发布到 Broker：
```
浏览器 WebSocket 发送: {"topic":"...","payload":"..."}
  → 网关通过 MQTT 发布
  → 回复: {"type":"mqtt_pub_ack","ok":true}
```

### 3. 静态文件服务（websocket_server.cpp）
**文件：** `src/services/web_services/websocket/websocket_server.cpp`

非 `/api/*` 的请求自动回退到静态文件服务，这样浏览器访问 `http://IP:8080/` 能直接显示前端页面。

**改动：**
- `on_event()` 中，API 未处理的请求调用 `mg_http_serve_dir()` 提供静态文件
- `HttpHandler` 返回类型改为 `bool`，true=已处理，false=交给静态文件

### 4. RestApi 依赖注入（rest_api.hpp + rest_api.cpp）
**新增文件：** `src/services/web_services/api/rest_api.cpp`

RestApi 新增静态成员 `registry_` 和 `mqtt_client_`，在 gateway_core.cpp 中注入。

### 5. HLS.js 占位文件
**新增文件：** `www/js/hls.min.js`

视频流是阶段五（可选）功能，先放占位文件避免页面报错。

### 6. CMakeLists.txt
**文件：** `CMakeLists.txt`

新增 `src/services/web_services/api/rest_api.cpp` 到编译列表。

## 二、你需要做什么

### 阶段二测试（先做这个）

#### 步骤 1：在虚拟机上安装 MQTT Broker

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install mosquitto mosquitto-clients

# 启动 Mosquitto
sudo systemctl start mosquitto
sudo systemctl enable mosquitto

# 验证是否运行
sudo systemctl status mosquitto
```

#### 步骤 2：修改配置文件

编辑 `config/environments/development.yaml`，把 `broker_host` 改成虚拟机 IP，端口使用当前对外监听的 `1884`：

```yaml
mqtt:
  enabled: true
  broker_host: "192.168.137.118"   # 虚拟机 IP，RK3568 网关连接这个 Broker
  broker_port: 1884
```

#### 步骤 3：编译运行

```bash
# 进入项目目录
cd ~/IotEdgeGateway

# 编译
./build.sh

# 运行
./build/iotgw_gateway
```

#### 步骤 4：测试阶段二 API

打开另一个终端，用 curl 测试：

```bash
# 测试 1：健康检查
curl http://192.168.137.170:8080/api/health
# 期望返回: {"status":"正常"}

# 测试 2：版本查询
curl http://192.168.137.170:8080/api/version
# 期望返回: {"version":"..."}

# 测试 3：设备列表（应该有 4 个传感器 + 3 个执行器）
curl http://192.168.137.170:8080/api/devices
# 期望返回: [{"id":"temp","kind":"sensor",...}, ...]

# 测试 4：单个设备详情
curl http://192.168.137.170:8080/api/devices/temp
# 期望返回: {"id":"temp","kind":"sensor",...}

# 测试 5：模拟 STM32 发送传感器数据（用 mosquitto_pub）
mosquitto_pub -h 127.0.0.1 -p 1884 -t "iotgw/dev/telemetry/temp" \
  -m '{"device_id":"temp","type":"sensor","data":{"value":25.5},"ts":1700000000}'

# 测试 6：再次查看设备详情，应该有数据了
curl http://192.168.137.170:8080/api/devices/temp
# 期望返回: status.last_payload 有数据

# 测试 7：查看设备状态聚合
curl http://192.168.137.170:8080/api/status
# 期望返回: {"temp":{"value":25.5},...}
```

#### 步骤 5：测试 MQTT 连接

```bash
# 订阅所有主题（另一个终端）
mosquitto_sub -h 127.0.0.1 -p 1884 -t "iotgw/dev/#"
```

## 五、常见问题

### Q1: 页面打不开
- 检查网关是否运行：`ps aux | grep iotgw`
- 检查端口是否监听：`netstat -tlnp | grep 8080`
- 检查防火墙：`sudo ufw allow 8080`

### Q2: MQTT 连接失败
- 检查 Mosquitto 是否运行：`sudo systemctl status mosquitto`
- 检查配置文件里的 broker_host 和 broker_port
- 手动测试 MQTT：`mosquitto_pub -h 127.0.0.1 -p 1884 -t test -m hello`

### Q3: 传感器数据不更新
- 检查 MQTT 主题是否正确：`iotgw/dev/telemetry/temp`
- 检查 JSON 格式是否正确
- 看网关日志有没有报错

### Q4: 控制命令没反应
- 检查 MQTT 是否连接：看网关日志
- 检查浏览器 F12 Network 标签，POST 请求是否成功
- 检查 `/api/control` 返回的状态码

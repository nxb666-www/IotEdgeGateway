# 部署说明

## 1. 部署目标

项目支持两种部署方式：

1. 部署包方式：适合开发调试、课程演示、快速验证。
2. Buildroot/overlay 方式：适合最终系统集成，烧录后开发板开机自带网关服务。

两种方式运行的核心程序一致：

```text
iotgw_gateway       网关主程序
mosquitto           MQTT Broker（系统安装）
```

默认端口：

```text
HTTP/WebSocket: 8081
MQTT Broker:    1883
```

## 2. 运行依赖

RK3568 运行环境需要具备：

| 依赖 | 用途 |
| --- | --- |
| libstdc++ | C++14 程序运行 |
| SQLite3 | 历史数据存储 |
| Mosquitto | MQTT Broker |
| FFmpeg | 摄像头预览、抓拍、录制 |
| GStreamer | 摄像头预览兜底方案 |
| V4L2 驱动 | 摄像头设备节点 `/dev/videoX` |
| 网络/WiFi | ESP8266 MQTT 接入、Web/Qt 访问 |

Mongoose 和 rapidyaml 由项目源码编译进程序，不需要在开发板上单独安装。

## 3. 构建

### 3.1 在虚拟机上交叉编译

在项目根目录执行：

```bash
./build.sh -a        # ARM64 debug 版本
./build.sh -a -r     # ARM64 release 版本（体积更小，strip 过）
```

产物位置：

```text
build/aarch64/debug/iotgw_gateway
```

### 3.2 生成部署包

```bash
./scripts/package_iotgw.sh
```

脚本会自动：
1. 查找编译产物（优先 aarch64/release，其次 aarch64/debug）
2. 拷贝可执行文件到 `iotgw_package/bin/`
3. 拷贝 `config/` 和 `www/` 目录
4. 生成 `start.sh`（支持 WiFi 自动连接）

生成目录结构：

```text
iotgw_package/
├── bin/
│   └── iotgw_gateway
├── config/
│   ├── environments/
│   │   ├── development.yaml
│   │   └── production.yaml
│   ├── devices/
│   │   ├── sensors.yaml
│   │   └── actuators.yaml
│   └── rules/
│       ├── automation-rules.yaml
│       └── alarm-rules.yaml
├── www/
│   └── index.html
├── data/
│   └── logs/
└── start.sh
```

## 4. 部署包方式

### 4.1 同步到 NFS 共享目录

在虚拟机上：

```bash
rsync -av --delete --no-perms --no-owner --no-group --exclude='.git' \
    /home/nie/IotEdgeGateway/ /home/nfs/IotEdgeGateway/
```

### 4.2 拷贝到开发板

在 RK3568 开发板上：

```bash
# 挂载 NFS
umount /mnt 2>/dev/null || true
mount -t nfs -o nolock,noresvport 192.168.137.118:/home/nfs /mnt

# 停止旧程序
killall iotgw_gateway 2>/dev/null || true
killall mosquitto 2>/dev/null || true

# 拷贝部署包
rm -rf /aiot/iotgw_package
cp -r /mnt/IotEdgeGateway/iotgw_package /aiot/iotgw_package
```

### 4.3 启动

```bash
cd /aiot/iotgw_package
chmod +x bin/* start.sh
./start.sh start
```

`start.sh` 会自动执行：

1. 检测 WiFi 连接，未连接则自动连接热点（默认 SSID: `nie`，密码: `12345678`）
2. 停止旧的 `iotgw_gateway` 和 `mosquitto`
3. 启动 Mosquitto MQTT Broker，监听 `0.0.0.0:1883`
4. 启动网关主程序，监听 `0.0.0.0:8081`
5. 日志写入 `data/logs/`

### 4.4 停止

```bash
cd /aiot/iotgw_package
./start.sh stop
```

### 4.5 重启

```bash
cd /aiot/iotgw_package
./start.sh restart
```

### 4.6 检查进程和端口

```bash
ps | grep -E 'iotgw|mosquitto'
netstat -tln | grep -E '1883|8081'
```

正常应看到：

```text
0.0.0.0:1883 LISTEN
0.0.0.0:8081 LISTEN
```

### 4.7 访问页面

查看开发板 IP：

```bash
ifconfig wlan0
```

浏览器访问：

```text
http://<RK3568_IP>:8081
```

## 5. 开发板网络配置

### 5.1 WiFi 自动连接

`start.sh` 内置 WiFi 自动连接逻辑，启动时会检测 `wlan0` 是否已连接，如果未连接则自动连接热点。

默认热点配置：

```text
SSID: nie
密码: 12345678
```

如需修改，启动时指定环境变量：

```bash
IOTGW_WIFI_SSID="新热点名" IOTGW_WIFI_PASS="新密码" ./start.sh start
```

### 5.2 有线/NFS 开发方式

开发阶段可通过 NFS 挂载虚拟机共享目录：

```bash
ip addr add 192.168.137.170/24 dev eth0 2>/dev/null
ip link set eth0 up
ping 192.168.137.118 -c 3
mount -t nfs -o nfsvers=3,nolock 192.168.137.118:/home/nfs /mnt
```

说明：

- NFS 只适合开发调试。
- 最终部署不依赖 `/mnt`。
- 最终应把部署包放到开发板本地 `/aiot/iotgw_package`。

## 6. 配置文件

开发配置：

```text
config/environments/development.yaml
```

生产配置：

```text
config/environments/production.yaml
```

关键配置（production.yaml）：

```yaml
name: IotEdgeGateway
env: production

paths:
  www_root: www
  log_file: data/logs/iotgw.log

logging:
  level: info

network:
  http_api:
    host: 0.0.0.0
    port: 8081

mqtt:
  enabled: true
  broker_host: "127.0.0.1"
  broker_port: 1883
  client_id: iotgw-prod
  topic_prefix: "iotgw/dev/"

storage:
  enabled: true
  db_path: "data/iotgw.db"
  retention_days: 7

control:
  manual_override_ms: 30000
```

## 7. 数据和日志位置

```text
/aiot/iotgw_package/data/iotgw.db           # SQLite 数据库
/aiot/iotgw_package/data/logs/iotgw.log      # 网关日志
/aiot/iotgw_package/data/logs/gateway.stdout.log  # 标准输出日志
```

Mosquitto 日志默认写入系统日志，可通过以下命令查看：

```bash
logread | grep mosquitto
```

## 8. STM32 通信配置

STM32 的 ESP8266 模块需要与 RK3568 在同一局域网，并连接到正确的 MQTT Broker。

配置文件：`F411/Hardware/ESP8266.h`

```c
#define ESP8266_WIFI_SSID      "nie"
#define ESP8266_WIFI_PASSWORD  "12345678"
#define ESP8266_MQTT_HOST      "192.168.233.107"   // 需要改为板子实际 IP
#define ESP8266_MQTT_PORT      1883
```

**重要**：`ESP8266_MQTT_HOST` 必须与 RK3568 开发板的实际 IP 一致。如果板子连接手机热点后 IP 变化，需要修改此宏并重新烧录 STM32 固件。

## 9. 功能验证

### 9.1 检查网关状态

```bash
wget -qO- http://127.0.0.1:8081/api/health
wget -qO- http://127.0.0.1:8081/api/version
```

### 9.2 检查传感器数据

```bash
wget -qO- http://127.0.0.1:8081/api/status
wget -qO- http://127.0.0.1:8081/api/devices
```

### 9.3 检查控制下发

```bash
# 开灯
wget -qO- --header="Content-Type: application/json" \
  --post-data='{"type":"control","payload":{"led_on":1,"led_br":80}}' \
  http://127.0.0.1:8081/api/control

# 关灯
wget -qO- --header="Content-Type: application/json" \
  --post-data='{"type":"control","payload":{"led_on":0}}' \
  http://127.0.0.1:8081/api/control

# 电机正转
wget -qO- --header="Content-Type: application/json" \
  --post-data='{"type":"control","payload":{"motor_on":1,"motor_sp":50,"motor_dir":0}}' \
  http://127.0.0.1:8081/api/control

# 蜂鸣器
wget -qO- --header="Content-Type: application/json" \
  --post-data='{"type":"control","payload":{"buzzer":1}}' \
  http://127.0.0.1:8081/api/control
```

### 9.4 检查 SQLite 历史数据

```bash
wget -qO- "http://127.0.0.1:8081/api/history?device_id=temp&limit=10"
```

### 9.5 检查日志

```bash
tail -f /aiot/iotgw_package/data/logs/iotgw.log
tail -f /aiot/iotgw_package/data/logs/broker.log
```

## 11. 常见问题

### 11.1 端口被占用

```text
HTTP 服务器启动失败
bind: 98
```

处理：

```bash
killall iotgw_gateway
killall mosquitto
./start.sh start
```

### 11.2 页面无数据

检查顺序：

1. `mosquitto` 是否运行：`ps | grep mosquitto`
2. `iotgw_gateway` 是否运行：`ps | grep iotgw`
3. ESP8266 MQTT 地址是否与板子 IP 一致
4. Broker 日志是否有客户端连接：`logread | grep mosquitto`
5. 网关日志是否收到消息：`cat data/logs/iotgw.log`

### 11.3 命令下发无效

检查顺序：

1. `/api/control` 是否返回成功
2. 网关日志是否发布 MQTT 消息
3. ESP8266 是否订阅 `iotgw/dev/cmd/#`
4. STM32 串口是否收到控制 JSON
5. `ESP8266_MQTT_HOST` 是否与板子实际 IP 一致

### 11.4 WiFi 连接失败

检查：

```bash
iwconfig wlan0
```

如果显示 `ESSID: off/any`，手动连接：

```bash
nmcli device wifi connect "nie" password "12345678"
```

### 11.5 NFS 挂载失败

检查虚拟机 NFS 服务是否启动：

```bash
# 虚拟机上
sudo systemctl status nfs-kernel-server
sudo exportfs -ra
```

### 11.6 编译报错：pthread 未定义

确保 CMakeLists.txt 中链接了 pthread：

```cmake
target_link_libraries(iotgw_gateway PRIVATE
    iotgw_common
    pthread
    dl
)
```

# 部署说明

## 1. 部署目标

项目支持两种部署方式：

1. 部署包方式：适合开发调试、课程演示、快速验证。
2. Buildroot/overlay 方式：适合最终系统集成，烧录后开发板开机自带网关服务。

两种方式运行的核心程序一致：

```text
iotgw_gateway       网关主程序
iotgw_mqtt_broker   本地 MQTT Broker
```

默认端口：

```text
HTTP/WebSocket: 8081
MQTT Broker:    1884
```

## 2. 运行依赖

RK3568 运行环境需要具备：

| 依赖 | 用途 |
| --- | --- |
| libstdc++ | C++14 程序运行 |
| SQLite3 | 历史数据存储 |
| FFmpeg | 摄像头预览、抓拍、录制 |
| GStreamer | 摄像头预览兜底方案 |
| V4L2 驱动 | 摄像头设备节点 `/dev/videoX` |
| 网络/WiFi | ESP8266 MQTT 接入、Web/Qt 访问 |

Mongoose 和 rapidyaml 由项目源码编译进程序，不需要在开发板上单独安装。

## 3. 构建

### 3.1 本地或虚拟机编译

在项目根目录执行：

```bash
./build.sh
```

常见产物：

```text
build/iotgw_gateway
build/iotgw_mqtt_broker
```

如果使用交叉编译脚本，产物可能位于：

```text
build/aarch64/debug/
build/aarch64/release/
```

以实际脚本输出为准。

### 3.2 生成部署包

```bash
sh scripts/package_iotgw.sh
```

生成目录：

```text
iotgw_package/
├── bin/
│   ├── iotgw_gateway
│   └── iotgw_mqtt_broker
├── config/
│   ├── environments/
│   ├── devices/
│   └── rules/
├── www/
│   └── index.html
├── data/
│   └── logs/
└── start.sh
```

说明：

- `bin/` 只放运行所需可执行文件。
- `config/` 放环境、设备、规则配置。
- `www/` 放 Web 前端资源。
- `data/` 放 SQLite 数据库和运行日志。
- `start.sh` 负责启动/停止 broker 和 gateway。

## 4. 部署包方式

### 4.1 拷贝到开发板

推荐目录：

```text
/opt/iotgw
```

示例：

```bash
mkdir -p /opt
cp -a iotgw_package /opt/iotgw
```

开发调试时也可以放在：

```text
/aiot/IotEdgeGateway
/mnt/IotEdgeGateway
```

但最终演示建议固定为 `/opt/iotgw`。

### 4.2 启动

```bash
cd /opt/iotgw
sh start.sh start
```

脚本会执行：

1. 停止旧的 `iotgw_gateway` 和 `iotgw_mqtt_broker`。
2. 启动本地 MQTT Broker，监听 `0.0.0.0:1884`。
3. 启动网关主程序，监听 `0.0.0.0:8081`。
4. 日志写入 `data/logs/`。

### 4.3 停止

```bash
cd /opt/iotgw
sh start.sh stop
```

### 4.4 重启

```bash
cd /opt/iotgw
sh start.sh restart
```

### 4.5 检查进程和端口

```bash
ps | grep iotgw
netstat -tln | grep -E '1884|8081'
```

正常应看到：

```text
0.0.0.0:1884 LISTEN
0.0.0.0:8081 LISTEN
```

### 4.6 访问页面

查看开发板 IP：

```bash
ifconfig wlan0
ifconfig eth0
```

浏览器访问：

```text
http://<RK3568_IP>:8081
```

示例：

```text
http://192.168.233.107:8081
```

## 5. 开发板网络配置

### 5.1 WiFi 热点方式

连接热点后获取 IP：

```bash
udhcpc -i wlan0
ifconfig wlan0
```

如果已经有 IP，则无需重复获取。

### 5.2 有线/NFS 开发方式

开发阶段可通过 NFS 挂载虚拟机共享目录：

```bash
ip addr add 192.168.137.170/24 dev eth0 2>/dev/null
ip link set eth0 up
ping 192.168.137.118 -c 3
mount -t nfs -o nfsvers=3,nolock 192.168.137.118:/home/nfs /mnt
```

说明：

- NFS 只适合开发调试。
- 最终部署不依赖 `/mnt`。
- 最终应把部署包放到开发板本地目录，或通过 Buildroot 集成进 rootfs。

## 6. 配置文件

开发配置：

```text
config/environments/development.yaml
```

生产配置：

```text
config/environments/production.yaml
```

关键配置：

```yaml
network:
  http_api:
    host: 0.0.0.0
    port: 8081

mqtt:
  broker_host: "127.0.0.1"
  broker_port: 1884
  topic_prefix: "iotgw/dev/"

camera:
  device: "/dev/video0"

storage:
  enabled: true
  db_path: "data/iotgw.db"
  retention_days: 7

control:
  manual_override_ms: 30000
```

## 7. 数据和日志位置

部署包方式：

```text
/opt/iotgw/data/iotgw.db
/opt/iotgw/data/logs/iotgw.log
/opt/iotgw/data/logs/broker.log
/opt/iotgw/data/logs/gateway.stdout.log
```

Buildroot/生产方式：

```text
/opt/iotgw/data/iotgw.db
/var/log/iotgw/iotgw.log
/var/log/iotgw/broker.log
/var/log/iotgw/gateway.stdout.log
```

视频临时帧：

```text
/tmp/rk_stream/live.jpg
```

拍照和录制文件：

```text
/opt/iotgw/www/snap_*.jpg
/opt/iotgw/www/rec_*.mp4
```

## 8. Buildroot/overlay 集成方式

### 8.1 目标

将项目作为 RK3568 rootfs 的一部分：

```text
/opt/iotgw/bin/iotgw_gateway
/opt/iotgw/bin/iotgw_mqtt_broker
/opt/iotgw/config/
/opt/iotgw/www/
/opt/iotgw/data/
/var/log/iotgw/
/etc/init.d/S99iotgw
```

烧录后开发板开机自动启动网关，不再依赖虚拟机、NFS 或手动复制源码。

### 8.2 Buildroot package

在 RK3568 SDK 的 Buildroot 中新增：

```text
buildroot/package/iotgw/Config.in
buildroot/package/iotgw/iotgw.mk
```

`iotgw.mk` 负责：

1. 指定项目源码目录。
2. 调用 CMake 交叉编译。
3. 安装 `iotgw_gateway` 和 `iotgw_mqtt_broker` 到 `/opt/iotgw/bin`。
4. 安装 `config` 和 `www` 到 `/opt/iotgw`。

### 8.3 overlay

overlay 负责放置：

```text
/etc/init.d/S99iotgw
/opt/iotgw/data/
/var/log/iotgw/
```

`S99iotgw` 主要逻辑：

```sh
/opt/iotgw/bin/iotgw_mqtt_broker 1884 > /var/log/iotgw/broker.log 2>&1 &
sleep 1
/opt/iotgw/bin/iotgw_gateway \
  --yaml-config /opt/iotgw/config/environments/production.yaml \
  --log-file /var/log/iotgw/iotgw.log \
  > /var/log/iotgw/gateway.stdout.log 2>&1 &
```

### 8.4 重新构建 rootfs

在 SDK 根目录执行：

```bash
./build.sh buildroot
```

如果要生成完整烧录镜像，根据 SDK 支持执行：

```bash
./build.sh firmware
```

或：

```bash
./build.sh updateimg
```

具体命令以当前 RK3568 SDK 的 `./build.sh -h` 为准。

## 9. 功能验证

### 9.1 检查网关状态

```bash
wget -qO- http://127.0.0.1:8081/api/health
wget -qO- http://127.0.0.1:8081/api/status
```

### 9.2 检查传感器数据

```bash
wget -qO- http://127.0.0.1:8081/api/devices/temp
wget -qO- http://127.0.0.1:8081/api/devices/humi
wget -qO- http://127.0.0.1:8081/api/devices/light
wget -qO- http://127.0.0.1:8081/api/devices/ir
```

### 9.3 检查控制下发

```bash
wget -qO- \
  --header="Content-Type: application/json" \
  --post-data='{"type":"control","payload":{"led_on":1,"led_br":80,"motor_on":0,"motor_sp":30,"motor_dir":0,"buzzer":0}}' \
  http://127.0.0.1:8081/api/control
```

然后观察 STM32 外设是否执行。

### 9.4 检查 SQLite 历史数据

```bash
wget -qO- "http://127.0.0.1:8081/api/history?device_id=temp&limit=10"
```

### 9.5 检查视频

```bash
wget -qO- http://127.0.0.1:8081/api/camera/probe
wget -qO- http://127.0.0.1:8081/api/camera/start_stream
wget -O /tmp/live.jpg http://127.0.0.1:8081/stream/live.jpg
```

## 10. 常见问题

### 10.1 端口被占用

现象：

```text
HTTP 服务器启动失败
bind: 98
```

处理：

```bash
killall iotgw_gateway
killall iotgw_mqtt_broker
netstat -tln | grep -E '1884|8081'
```

再重新启动。

### 10.2 页面无数据

检查顺序：

1. `iotgw_mqtt_broker` 是否运行。
2. `iotgw_gateway` 是否运行。
3. ESP8266 是否连接到 RK3568 的 `1884`。
4. Broker 日志是否出现 `PUB topic=iotgw/dev/telemetry/...`。
5. `/api/devices/temp` 是否有 `last_payload`。

### 10.3 命令下发无效

检查顺序：

1. `/api/control` 是否返回成功。
2. Broker 日志是否出现 `PUB topic=iotgw/dev/cmd/control`。
3. ESP8266 是否订阅 `iotgw/dev/cmd/#`。
4. ESP8266 串口是否向 STM32 输出控制 JSON。
5. STM32 是否解析并执行控制字段。

### 10.4 规则覆盖手动控制

检查配置：

```yaml
control:
  manual_override_ms: 30000
```

如果设置太短，风扇或蜂鸣器可能刚手动打开又被规则关闭。

### 10.5 摄像头无法启动

检查：

```bash
ls -l /dev/video*
wget -qO- http://127.0.0.1:8081/api/camera/probe
tail -n 80 stream/ffmpeg.log
tail -n 80 stream/gstreamer.log
```

如果内核日志出现：

```text
No link between dphy and sensor
Not active sensor
Unexpected sensor id
```

说明问题在摄像头排线、sensor 驱动、设备树或 MIPI 链路，不是 Web 页面问题。

### 10.6 Buildroot 同步源码失败

如果 Buildroot 同步项目目录时报 `.ssh_tmp`、`.git`、`iotgw_package` 权限或无关文件错误，应使用干净源码目录参与构建，只保留：

```text
CMakeLists.txt
cmake/
src/
config/
www/
```

不要把密钥、NFS 临时文件、构建产物、`.git` 放进 Buildroot package 源码目录。

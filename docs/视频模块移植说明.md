# 视频模块移植说明

## 1. 模块目标

视频模块用于在 Web/Qt 端查看 RK3568 本地摄像头画面，并支持拍照和录制。该模块不经过 STM32，也不走 MQTT，属于 RK3568 本地外设能力。

当前视频链路：

```text
CSI/USB Camera
  -> /dev/videoX
  -> V4L2
  -> FFmpeg 优先 / GStreamer 兜底
  -> /tmp/rk_stream/live.jpg
  -> Mongoose HTTP
  -> Web/Qt 显示
```

## 2. 已实现接口

```text
GET /api/camera/probe
GET /api/camera/start_stream
GET /api/camera/stop_stream
GET /api/camera/snapshot
GET /api/camera/start_record
GET /api/camera/stop_record
GET /stream/live.jpg
```

对应文件：

```text
src/services/web_services/api/camera_api.hpp
src/services/web_services/api/camera_api.cpp
```

路由入口：

```text
src/services/web_services/api/rest_api.cpp
```

前端入口：

```text
www/index.html
```

## 3. 实现方式

启动预览时，后端会优先尝试 FFmpeg：

```text
ffmpeg -f v4l2 -i /dev/videoX -vf scale=640:-1,fps=10 -update 1 /tmp/rk_stream/live.jpg
```

如果 FFmpeg 没能生成有效 JPEG，会尝试 GStreamer：

```text
gst-launch-1.0 v4l2src device=/dev/videoX ! videoconvert ! jpegenc ! filesink
```

Web 页面通过刷新下面接口显示画面：

```text
/stream/live.jpg?t=时间戳
```

## 4. 设备选择

默认配置：

```yaml
camera:
  device: "/dev/video0"
```

程序会优先使用配置设备，同时扫描 `/dev/video0` 到 `/dev/video15`，选择名称中包含 `mainpath`、`selfpath`、`USB`、`uvc`、`camera` 的节点。

查看当前视频节点：

```bash
ls -l /dev/video*
```

查看节点信息：

```bash
v4l2-ctl -d /dev/video0 --all
v4l2-ctl -d /dev/video0 --list-formats-ext
```

## 7. 常见问题

### 7.1 有 `/dev/video0` 但无法取流

`/dev/video0` 存在只说明系统创建了 V4L2 节点，不代表摄像头 sensor 已经真正工作。需要继续确认：

```text
sensor 是否识别
I2C 是否能读到设备
MIPI DPHY 是否建立链路
RKISP 是否处于 active 状态
```

典型错误：

```text
No link between dphy and sensor
Not active sensor
Unexpected sensor id(000000)
```

这类问题优先检查排线方向、sensor 驱动、设备树和内核日志。

### 7.2 OV5695 识别成功标志

成功时内核日志应出现：

```text
ov5695 2-0036: Detected OV005695 sensor
```

并且驱动链接存在：

```bash
readlink /sys/bus/i2c/devices/2-0036/driver
```

### 7.3 没有画面

查看日志：

```bash
tail -n 80 stream/ffmpeg.log
tail -n 80 stream/gstreamer.log
```

确认开发板 rootfs 中有：

```bash
ffmpeg -version
gst-launch-1.0 --version
```

### 7.4 录制失败

录制依赖预览帧，所以需要先调用：

```text
GET /api/camera/start_stream
```

再调用：

```text
GET /api/camera/start_record
GET /api/camera/stop_record
```

# 视频模块移植说明

## 来源

参考目录：

```text
C:\Users\lenovo\Desktop\小组\视频推流
```

原 demo 是独立的 C/Mongoose 服务器，包含：

- HTTP 静态页面
- 内置 MQTT Broker
- 摄像头推流、抓拍、录像接口

本项目已经有独立 Mosquitto Broker 和 `iotgw_gateway`，所以没有移植原 demo 的内置 MQTT Broker，也没有改 HTTP 端口。

## 实现方式

当前使用 `ffmpeg` 从 RK3568 摄像头设备采集 JPEG 预览帧：

```text
/dev/video9 -> ffmpeg -> /tmp/rk_stream/live.jpg
```

前端不再使用 HLS，而是定时刷新：

```text
/stream/live.jpg?t=时间戳
```

这样和小组视频 demo 当前可用实现一致，依赖更少，适合作为第一版移植。

## 常见问题

### 启动推流失败

检查开发板是否有摄像头设备：

```bash
ls -l /dev/video*
```

当前默认设备是：

```text
/dev/video9
```

如果实际不是 `/dev/video9`，需要把 `camera_api.cpp` 里的 `kVideoDevice` 改成实际设备。

### 没有画面

查看日志：

```bash
tail -n 80 stream/ffmpeg.log
```

并确认开发板安装了 `ffmpeg`。
